# 🍚 밥세권 (BabSegwon)

**잔여 음식 픽업 기반 웹 플랫폼**

> 음식점의 잔여 재고를 합리적인 가격으로 제공하고,
> 사용자는 합리적인 소비를, 가게는 폐기 비용 절감을 달성하는 픽업 기반 서비스

---

## 📌 프로젝트 개요

**밥세권**은
✔️ 음식점의 **마감 전 잔여 음식 문제**
✔️ 소비자의 **합리적 소비 니즈**
를 동시에 해결하기 위해 기획된 **픽업 기반 웹 서비스**입니다.

지역 기반 배달이 아닌,
**결제 → 예약 → 픽업** 중심 구조로 설계하여
실제 소규모 음식점 운영 환경에 맞춘 서비스를 구현했습니다.

---

## 🎯 기획 배경 (데이터 기반 접근)

### 문제 인식

* 소규모 음식점은 마감 후 **잔여 재고 폐기 비용** 부담
* 소비자는 합리적인 가격의 식사 기회 부족
* 배달 중심 서비스는 수수료·운영 부담이 큼

### 기획 방향

* **배달 제거 + 픽업 전용 구조**
* 음식점 운영 부담 최소화
* 사용자 결제 안정성 및 주문 무결성 확보

---

## 🧩 핵심 서비스 구조

```
상품 등록 → 잔여 수량 관리 → 사용자 예약
→ Toss Payments 결제 → 주문 확정
→ 매장 픽업 → 관리자 확인
```

* 주문 스냅샷 기반 데이터 정합성 유지
* 결제 승인/취소 Webhook 연동
* 관리자(Admin) 운영 페이지 포함

---

## 🗂️ 데이터베이스 설계 (ERD)

* Oracle 기반 관계형 DB 설계
* 핵심 테이블

  * `TBL_MEMBER`
  * `TBL_STORE`
  * `TBL_ITEM`
  * `TBL_CART`
  * `TBL_ORDERS`
* 무결성 제약 조건 및 관계 중심 설계
* 실제 DML(insert/update/delete/select) 테스트 수행

## ✨ 느낀 점

이번 **밥세권 프로젝트**를 통해
단순 기능 구현이 아닌 **실제 운영 환경을 고려한 서비스 설계**의 중요성을 체감했습니다.

특히

* 결제 무결성
* 주문 데이터 정합성
* 관리자 운영 관점
  을 함께 고려해야 서비스가 안정적으로 동작한다는 점을 배웠습니다.
